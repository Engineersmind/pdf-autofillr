## Comparison: Local vs AWS Lambda

### Local Deployment
```
Source:      /app/data/input/xxx.pdf
             ↓ COPY
Processing:  /tmp/processing/xxx_input.pdf
             ↓ EXTRACT, MAP, EMBED, FILL
Processing:  /tmp/processing/xxx_filled.pdf
             ↓ COPY
Output:      /app/data/output/xxx_filled.pdf
             ↓ CLEANUP
Processing:  (deleted)
```

### AWS Lambda Deployment
```
Source:      s3://bucket/input/xxx.pdf
             ↓ DOWNLOAD
Processing:  /tmp/processing/xxx_input.pdf
             ↓ EXTRACT, MAP, EMBED, FILL (SAME CODE!)
Processing:  /tmp/processing/xxx_filled.pdf
             ↓ UPLOAD
Output:      s3://bucket/output/xxx_filled.pdf
             ↓ CLEANUP (SAME!)
Processing:  (deleted)
```

**Only Difference: COPY vs DOWNLOAD/UPLOAD**

## Config.ini Structure

```
config.ini
├── [general]
│   └── source_type = local | aws | azure | gcp
│
├── [local]
│   ├── input_base_path = /app/data/input
│   ├── output_base_path = /app/data/output
│   └── processing_dir = /tmp/processing
│
├── [aws]
│   ├── bucket_name = my-bucket
│   ├── input_prefix = input/
│   └── output_prefix = output/
│
├── [azure]
│   └── (similar to aws)
│
├── [gcp]
│   └── (similar to aws)
│
└── [file_naming]
    ├── input_pdf_pattern = {user_id}_{session_id}_{pdf_doc_id}.pdf
    ├── filled_pdf = {user_id}_{session_id}_{pdf_doc_id}_filled.pdf
    └── (all other patterns)
```

## Event Flow

```
1. SDK → API Request
   POST /mapper/make-embed-file
   {
     "user_id": 553,
     "session_id": "086d...",
     "pdf_doc_id": 990
   }
   
2. API Server → Entrypoint
   entrypoints/local.handle_local_event(event)
   
3. Entrypoint → File Ops
   - Read config.ini
   - Build paths
   - Copy files: source → processing
   
4. Entrypoint → Operations
   operations.handle_make_embed_file_operation(config, ...)
   
5. Operations → Processing
   - extract() → /tmp/processing/xxx_extracted.json
   - map() → /tmp/processing/xxx_mapped.json
   - embed() → /tmp/processing/xxx_embedded.pdf
   
6. Entrypoint → File Ops
   - Copy results: processing → source
   - Cleanup: delete /tmp/processing/*
   
7. API Server → SDK Response
   {
     "status": "success",
     "output_paths": {
       "filled_pdf": "/app/data/output/xxx_filled.pdf"
     }
   }
```

## Key Principle

```
┌────────────────────────────────────────────┐
│  ENTRYPOINTS (Source-specific)             │
│  - Handle I/O (copy/download/upload)       │
│  - Different per source                     │
└────────────────────────────────────────────┘
                  │
                  ▼
┌────────────────────────────────────────────┐
│  OPERATIONS (Source-agnostic)              │
│  - Handle processing (extract/map/fill)    │
│  - SAME for all sources                     │
└────────────────────────────────────────────┘
```

**Separation of Concerns = Maintainability!**
